2. New Changes (Not Present in Documents)
------------------------------------------
Features present in the code but not mentioned in the PDFs.

* Mobile App:
  - Caretaker (Guardian) Patient Selection Switcher: Monitored patients dropdown on the dashboard top bar (home_screen.dart).
  - Caretaker Query Interceptor: DioClient automatically appends patientId parameter on all requests when a patient profile is active.
  - Guardian Read-Only Locks: Forms disable add/edit options when a guardian is browsing.
  - Manage Connected Hospitals Access Screen: connected_hospitals_screen.dart view list and revoke button.
  - Bypass Authentication developer action: login bypass in login_screen.dart, splash token-skip rules.
  - Firebase Cloud Messaging integration: notification_manager.dart setups.

* Admin Panel:
  - Roster of patient compliance metrics (average, missed, high-risk flags) inside ComplianceTracker.jsx.
  - Deleted Selfies Log: View auto-deleted or admin-deleted selfies with instant recover actions.
  - Patient vital curves viewer (Vitals.jsx).
  - Admin authentication screen (Login.jsx).
  - Medication schedules manager for patient records.

* Backend:
  - Hospital Isolation Query Rules: Enforced in user list queries.
  - Connected Hospital OTP flow routes (/hospitals/send-link-otp, /hospitals/verify-link).
  - Guardian invite mappings & caretaker invitations (caretakerInviteModel.js, emailHelper.js).
  - Guardian middleware blocker (blockGuardianWrite checks).
  - Twilio SMS helper classes (smsHelper.js).
  - Admin selfie recovery endpoints.
  - FCM Server SDK setup and notifications dispatchers.
  - Adherence dashboard calculator query.

3. Modified Features
---------------------
Features implemented differently from original requirements.
- Navigation Consolidated: Combine separate Reports & Vitals tabs into Health Hub to simplify layout.
- Premium Photo Expiry Cap: Imposed a 3-month storage limit on premium verifications rather than storing them indefinitely.
- Split Reports Files: Reports and Prescriptions are managed as separate collections and list screens rather than a single database collection.

4. Missing Features
--------------------
- Visual stamp of date/time on selfie photos.
- Medication Compliance Adherence Chart in Mobile App.
- WASA Compliance validation.
